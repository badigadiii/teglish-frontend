# Media Domain

## Назначение

Домен `media` обслуживает загрузку и выдачу аудио- и видеофайлов, которые используются прежде всего в `dictation` упражнениях.

## Файлы

- `src/media/models.py`
- `src/media/schemas.py`
- `src/media/repository.py`
- `src/media/service.py`
- `src/media/router.py`
- `src/media/storage.py`

## Модель данных

Таблица `dictation_media_files` хранит:

- `id`
- `author_id`
- `filename`
- `extension`

Физический путь к файлу в БД не хранится. Он вычисляется storage-слоем.

## API

### `POST /media`

Загружает файл.

Требования:

- только авторизованный пользователь;
- поддерживаются только `audio/*` и `video/*`;
- у файла должно быть расширение.

Опционально можно передать `name`. Оно нормализуется:

- кириллица транслитерируется;
- небуквенно-цифровые символы заменяются на `_`;
- к имени добавляется UUID suffix.

В ответе backend возвращает:

- `author_id`
- `media_filename`
- `extension`
- `media_url`

### `GET /media/{media_filename}`

Возвращает файл по имени, используя данные из БД и storage-слой.

Если запись есть в БД, но физический файл удален, API возвращает `404`.

### `DELETE /media/{media_filename}`

Удаляет запись о медиа и пытается удалить физический файл.

Доступ:

- владелец файла;
- `admin`;
- `moderator`.

Если физический файл уже отсутствует, но запись в БД есть, удаление все равно считается успешным и возвращает `204`.

## Storage abstraction

`src/media/storage.py` содержит:

- `MediaStorage` - абстрактный интерфейс;
- `LocalMediaStorage` - текущая реализация поверх локальной файловой системы;
- `build_media_filename()` и нормализацию имени.

Сейчас используется:

- локальная папка `uploads`;
- `FileResponse` для отдачи контента.

## Важное архитектурное замечание

`MediaStorage` уже выделен как абстракция, поэтому этот домен явно подготовлен к будущей миграции на S3-совместимое хранилище. В документации проекта стоит считать это ожидаемым направлением развития: локальное хранение является текущей реализацией, а не окончательной архитектурой.

При переходе на S3, скорее всего, изменятся:

- реализация `get_media_storage()`;
- конкретный класс storage вместо `LocalMediaStorage`;
- политика именования, presigned URL или проксирование через backend.

При этом слой `MediaService` и интеграция с `exercises` могут остаться почти без изменений.
