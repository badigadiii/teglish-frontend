# Users Domain

## Назначение

Домен `users` отвечает за хранение и чтение пользовательских данных, а также за регистрацию новых пользователей через сервисный слой.

## Файлы

- `src/users/models.py`
- `src/users/schemas.py`
- `src/users/repository.py`
- `src/users/service.py`
- `src/users/router.py`

## Бизнес-логика

`UsersService` сейчас делает немного:

- ищет пользователя по `username`;
- регистрирует пользователя;
- хеширует пароль перед сохранением;
- преобразует `IntegrityError` в `409 Username already registered`.

Если `name` не передан, сервис использует `username` как отображаемое имя.

## API

### `GET /users/me`

Возвращает текущего активного пользователя.

Доступ:

- только по валидному Bearer token;
- неактивный пользователь получает `400 Inactive user`.

### `GET /users/me/media`

Возвращает paginated-список медиа, загруженных текущим пользователем.

Доступ:

- только по валидному Bearer token.

Формат ответа:

- `items`
- `total`
- `page`
- `size`
- `pages`

Каждый элемент `items` содержит:

- `author_id`
- `media_filename`
- `extension`
- `media_url`

## Замечания по развитию

Сейчас `users` - базовый домен без профиля обучения, рейтинга, статистики и истории прохождения упражнений. Для реализации продуктового видения сюда со временем, вероятно, добавятся:

- очки и рейтинг;
- история попыток;
- роли модерации и дополнительный workflow вокруг пользовательского контента.
